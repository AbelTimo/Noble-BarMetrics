# Claude MD Rules

## File Organization
- Use descriptive file and folder names that clearly indicate purpose
- Maintain a consistent directory structure across the project
- Keep related files together in logical groupings

## Code Quality
- Write clear, maintainable code with meaningful variable and function names
- Add comments for complex logic and non-obvious implementations
- Follow the language-specific style conventions and best practices
- Keep functions focused on a single responsibility

## Documentation
- Document public APIs and exported functions
- Include usage examples for non-trivial functionality
- Update documentation when making changes to existing code
- Use clear, concise language in comments and documentation

## Git Practices
- Make atomic commits with clear, descriptive messages
- Reference relevant issues or tickets in commit messages
- Keep commits focused on a single logical change
- Review code before committing to catch errors early

## Testing
- Write tests for critical functionality
- Maintain reasonable test coverage
- Use descriptive test names that explain what is being tested
- Keep tests focused and independent

## Performance
- Be mindful of performance implications when adding new code
- Optimize algorithms and database queries for efficiency
- Monitor and profile code to identify bottlenecks
- Document any performance-critical sections

## Security
- Follow secure coding practices
- Never hardcode secrets or credentials
- Validate and sanitize user input
- Keep dependencies updated for security patches

## Collaboration
- Communicate clearly with team members about changes
- Ask for code reviews before merging significant changes
- Be open to feedback and suggestions
- Help others understand your code through documentation
